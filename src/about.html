About the Project

Why this dashboard exists

Cosmetics brands generate large volumes of transactional data — but that data rarely reaches decision-makers in a form they can act on quickly. This project simulates a real-world data analyst task: turning raw cosmetics sales data into a single, interactive dashboard that answers the questions a marketing or category team asks every week.

Objective

Build a Tableau dashboard that reveals which categories and brands drive revenue, how demand trends over time, which regions matter most, and how customers rate what they buy — all in one filterable view.

Process

Ideation & Requirement Analysis

Defined the problem statement, target audience, and the exact fields needed from the dataset.

Design

Planned a five-chart dashboard layout and a consistent visual language across categories.

Development

Built each worksheet in Tableau Public, then combined them into one interactive dashboard with cross-filtering.

Testing & Documentation

Validated filters, sorting, and data accuracy, then documented findings for this site and the project report.

Tech Stack

Tableau Public

Dashboard design and data visualization.

Python / Flask

Lightweight app serving this companion site.

HTML, CSS, JS

Static pages for dashboard, story, and dataset views.

GitHub

Version control and project hosting.
